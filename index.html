Brisbane Housing Affordability Analysis

Brisbane's housing supply is struggling to keep pace with rapid population growth — but the pressure isn't uniform across suburbs. This project combined Queensland Spatial Catalogue geospatial data, ABS population projections, and residential land supply data to map dwelling shortages across 33 Brisbane SA2 regions. 571 land parcels were aggregated to SA2 level and merged with population projections through 2046. A Random Forest Classifier (83% accuracy) labelled each region as Low, Medium, or High supply, while a Random Forest Regressor (R²=0.489) estimated dwelling yield — revealing where housing demand is most likely to outpace availability.

Nashville Housing Data Cleaning — SQL

Raw real estate data is rarely clean enough to analyse directly. This project took a messy Nashville housing dataset and transformed it into a fully structured, analysis-ready format entirely in PostgreSQL — populating missing property addresses via self-join, splitting combined address fields into discrete columns, standardising inconsistent values, and removing duplicate records using window functions and CTEs. It demonstrates the kind of rigorous SQL data engineering that underpins any reliable analytics pipeline. This project demonstrates the data engineering discipline that sits beneath every reliable analytics pipeline — without clean, consistent data, even the most sophisticated models produce unreliable outputs. The techniques applied here (deduplication, self-joins, window functions, NULL handling) map directly to the first month's work in most data analyst roles.

PostgreSQL SQL Window Functions CTEs Data Cleaning

Calories Burnt Prediction

Fitness trackers estimate calorie burn — but how accurate are they really? This project built and compared five machine learning models (Linear Regression, Lasso, Ridge, Random Forest, and XGBoost) to predict calories burned during exercise using physiological and activity data from 15,000 workout sessions. XGBoost delivered the best results, achieving an R² of 0.9990 and MAE of 1.38 kcal, while the comparison also exposed clear overfitting in Random Forest — a key insight for responsible model selection. The model is accurate enough to power real-time feedback loops in wearables applications or personalised health coaching platforms.

Movie Data Analysis & Visualisation

What actually drives a movie's box office success — star power, critic scores, or sheer budget? This exploratory analysis examined 7,668 movies to identify which features most strongly predict gross earnings. Using regression plots, annotated heatmaps, correlation analysis, and an interactive Tableau dashboard, the project found that budget is the single strongest predictor (correlation of 0.74), followed by audience votes (0.63). The analysis tells a clear, evidence-based story about the economics of the film industry that a non-technical stakeholder could immediately act on. These findings are directly actionable for studio acquisition teams and streaming platforms deciding where to allocate production investment — quantifying the return premium for high-budget productions and the relative fragility of critic-dependent releases in a market increasingly driven by audience scores.

Python Pandas Matplotlib Seaborn EDA Correlation Analysis Tableau

Movie Recommendation System

Recommendation engines power Netflix, Spotify, and Amazon — but how do they actually work? This project built a content-based movie recommender from scratch using the TMDB 5,000 dataset. Each film was encoded as a text vector combining genres, cast, director, and plot keywords, then cosine similarity was used to find the closest thematic matches. Searching for "Avatar" surfaces Guardians of the Galaxy and Star Trek — showing the system has learned genuine semantic relationships between films. Collaborative filtering via SVD was also explored in the research notebook as a complementary approach.
